# Security Reporting Quick Guide

## How to Report Security Vulnerabilities

### For Security Issues (Private):
1. Go to the "Security" tab in this GitHub repository
2. Click "Report a vulnerability" 
3. Fill out the security advisory form with:
   - Vulnerability description
   - Steps to reproduce
   - Impact assessment
   - Suggested fixes (if any)

### For General Security Questions (Public):
- Create a regular GitHub issue
- Use the "question" or "security" label
- These are for non-sensitive security-related discussions

### For Code of Conduct Issues:
- Create a private issue or contact repository maintainers
- Use GitHub's reporting features
- Mention @garisonmike in the issue

## Communication Channels

All project communication happens through GitHub:
- **Issues**: Bug reports, feature requests, general questions
- **Discussions**: Community interaction and Q&A
- **Security Advisories**: Private vulnerability reporting
- **Pull Requests**: Code collaboration and reviews

No external email addresses or services are used for project communication.
